Reusable code and used technologies

  1. Server:

    1. Server itself: Apache httpd

      Protocol: HTTPS.

      Additional modules:

      • Cryptography: mod_nss
      • Security: mod_security
      • Gateway Interface: mod_wsgi

    2. Webservice framework: Ladon

      Protocols: SOAP and JSON-WSP (chosen in configuration).

      Description in WSDL.

    3. Database connection: psycopg2

    4. Object relational mapper: SQLAlchemy

  2. Database management systems:

    1. Database of metadata: PostgreSQL

      It’s built with: CPython 3.4, NSS + nss_compat_ossl, ICU.

    2. Column-oriented database: ??

      First idea was to use MonetDB and foreign data wrapper monetdb_fdw to embed that into PostgreSQL. Flaws:

      • License of MonetDB is not compatible with GPL (it is MPL 1.1 with a choice of law). See discussion.
      • It is not clear whether foreign data wrappers share memory with PostgreSQL or copies values. The latter should be avoided.

  3. Client:

    1. Webservice client: TODO

    2. GUI:

      • PyQt
      • QtAlchemy ?

    3. Plotting: Matplotlib

  4. Cryptographic library: Mozilla NSS

    • License of OpenSSL is incompatible with GPL
    • NSS is one the most featured cryptographic library
    • NSS is FIPS 140-2 certified
    • OpenSSL compatibility library: nss_compat_ossl

    Comparisons of cryptographic libraries:

    • http://curl.haxx.se/docs/ssl-compared.html
    • http://en.wikipedia.org/wiki/Comparison_of_TLS_implementations

  5. Sending of time series from the server to the client (for plotting):

    • Protocols: UDT (by default) and TCP (reserved)
    • Server: GridFTP

  6. Mathematical, numerical and statistical libraries:

    (list is for the beginning)

    • bottleneck
    • blas/lapack/atlas implementation
    • NumPy
    • SciPy
    • Pandas
    • StatsModels
    • Numexpr
    • TA-Lib

  7. Common libraries:

    • zlib

    • libxml2

    • libxslt

    • ICU

    • Python packages:

      • six
      • pytz
      • python-dateutil
